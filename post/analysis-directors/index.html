Analysis of Director Genders in Popular Films

Feb 25, 2024 7 minutes

Check out the Github for this project

Check out the Shiny app

Overview

After a bit of recent uproar around the director for the Barbie film not recieving an oscar award, I decided that it might be interesting to do a project analysing the disparity between genders of directors in popular films.

When deciding which dataset to choose, I checked out a few different film sites and their rating systems. I settled on Letterboxd because it seemed to be the site with the highest number of user watches and reviews for most films compared to sites like IMDB and TMDB. I also wanted to get a dataset that was representative of popular films in the general public, and I think the film ratings on IMDB and TMDB are more influenced by film critic reviews.

Letterboxd has the option to filter by rating or popularity. I went with popularity, because filtering by high ratings gives many films that have not been watched by a lot of people, but still have a high average rating. Letterboxd popularity score is calculated by aggregating a range of features from the Letterboxd site such as number of ratings, number of users who reported watching the film, and number of Letterboxd playlists the film has been added to. It is a pretty good indiciation of a films overall popularity with the general public.

I decided to go with the top 100 most popular films each year on Letterboxd from 1980 - 2023.

Scraping Film Data

The data scraping Python script can be found in the Github for this project in letterboxd_scraper.ipynb

Letterboxd unfortunately does not have a publically accessible API, so I decided to create a script in python that opened the Letterboxd film list page for each year filtered by popularity and scraped the information I needed for each film. The gender of the film directors was not listed on the Letterboxd page, but there was an external link to the TMDB page for each director where I scraped the gender and birthdate info from.

I used the Python library Selenium for my web scraping. Selenium is a library that allows you to automate browser activity such as clicking links and scrolling down, as well as pulling data from the page html. It was an interesting and sometimes frustrating task, which taught me a lot about web scraping. The main lesson was to try and catch as many errors as possible, so that you don’t have to keep restarting your script. This was especially important for a script like this that took about 5 - 10 seconds to get all the information for each film, and a few hours in total.

The most common errors were thrown when the script was searching for an html element e.g. director name, and the page had not loaded that element yet. I tried setting the driver settings so that the page would always wait to load completely before moving on, but this slowed down the script immensely, and often froze when the browser got stuck loading a page indefinitely.

The best way that I found to avoid this was to set the driver settings to not wait for the page to load at all, and put all pieces of code searching for web html elements in a function that waits 10 seconds for them to exist. This way, it would only ever wait as long as needed for the target element to load. I also placed each of these element searching lines inside a try catch block, so that the script would continue if an element was not found after 10 second. For elements that were definitely there such as film title, I placed the try block inside a loop that would restart the driver and try again indefinitely until the element was found. I also had an issue where the driver would start glitching after running for a while and just wouldn’t find web elements that I knew were definitely there, so I set the driver to reset every 20 films, which fixed this.

Even with all this, I still had the driver throw some random new error every now and then, but it only happened two or three times for all of the data. I set up the script so that all previously collected data was saved and I only had to change the starting film year to continue the script, so this was acceptable.

Cleaning Film Data

The data cleaning Python script can be found in the Github for this project in cleaning_film_data.ipynb

The data pulled from my script wasn’t in the best format. The film rating text that was scraped from the site was a dict for each film with this format for each star rating - ‘0.5’: ‘1,647\xa0half-★ ratings (0%)’. The Directors for each film (sometimes 2 or more) were stored in a list of lists with each list element in this format ‘Steven Spielberg’, ‘Gender\nMale’, ‘Birthday\nDecember 18, 1946 (77 years old)’. Each film also had multiple Genres stored in a list.

Here’s an step by step overview of the data cleaning process:

  • Expanding the ratings out into multiple columns for each rating (0.5 -> 5.0) and calculating the total number of ratings and average rating from these numbers.
  • Creating a new dataframe with a unique list of directors from all the director lists for each film and cleaning up the birthday and gender text.
  • Cleaning up the directors column in the original dataframe to just be a list of names.
  • Adding Age (at film release year) and Gender columns in main dataframe matching column with the director names column.
  • Adding boolean columns for “film has x gender director”, for each of the director genders.
  • Cleaning up Genre column, removing some of the weird genre names and replacing with main genre category names.

I also created a few basic plots in this Python notebook just to get an overview of the data.

Visualising Data with R and Shiny

The Shiny app code can be found in the Github for this project in the shiny-app folder.

I decided to use Shiny with R to create an interactive visualisation app for my data.

I started by writing a short R script that would convert my csv files exported from my Python notebook into R dataframe rds files. There is a library called Feather that can do this, but this library didn’t seem to handle converting the list objects in my dataframe very well, so I decided to do it myself, by writing simple functions that parse all the csv text into various R data types.

The creation of the UI and server logic was pretty straight forward. You can find this in app.R in the shiny-app folder. The hardest part about building this app was working out the best ways to structure my data for plotting. Since many of the columns contained lists, I had to expand these lists out whenever I wanted to draw insights.

I ended up creating a few dataframes with different structures for plotting. The first, named films_expanded_genders, which stacked all rows that were true for “film has x gender director” on top of each other, so that I could plot the amount of each film that had x gender directors. This was not possible when there was overlap in the boolean columns (because some films had multiple directors).

The second, named director_film_count, expanded out the director names list column from the main films dataframe so that I could count how many films each director had directed.

The third and fourth, named female_genres and male_genres, expanded out the genres columns and counted them up for films with female or male directors.

There were 3 non-binary directors in this data, which I left out of the plots as the number is too small to draw any useful statistical insights. There was also 62 directors over 23 films whos gender information was not listed on TMDB. A brief look at these directors showed that they were mostly the last 5 or so less well known directors listed in a large group of directors for a TV series, so this is unlikely to have skewed the data much.

Once this was all set up, I had the fun task of making various plots, and writing some inline CSS to make it a little more visually appealing.

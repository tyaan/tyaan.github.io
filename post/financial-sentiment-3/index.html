Sentiment Analysis of Financial News Headlines with Python - Part 3

Feb 18, 2024 8 minutes

Check out the Github for this project

This post is the first of a 3 part series.

  • Part 1 - Scraping financial news headlines.
  • Part 2 - Performing sentiment analysis to get sentiment scores for each headline.
  • Part 3 - Aggregating our headline sentiment scores to create a sentiment value for each day the market is open, and checking for correlation between this and the stock price movement over the following day.

This project is the first addition to my data science portfolio. The purpose of this project is to showcase web scraping, sentiment analysis, and some basic statistical analysis. I’m going to write it in a tutorial style if anyone out there wants to follow along.

In this part we will be aggregating our AAPL stock news headline sentiment scores to get a sentiment value at the close of each stock market day. We will then check if there is any correlation between these scores and the price change at the close of the following market day. If the sentiment in the news headlines is any indication of price movement, you would expect to see positive price movement on days with high positive sentiment, and negative price movement on days with high negative sentiment.

First we need to collect daily AAPL stock price data. You can download this data as csv files from marketwatch.

Now lets combine our multiple AAPL stock price csv files downloaded from marketwatch into one data frame. We will also load our article headlines from the csv file we created in Part 2.

import pandas as pd
import os
from datetime import datetime

stock_price_folder = "AAPL stock price data"

df_list = []
# Loops over all csv files in the stock price folder
for root, dir, files in os.walk(stock_price_folder):
    for file in files:

        print(file)

        # Adds the csv file to a data frame and adds that data frame to a list of data frames
        df_list.append(pd.read_csv(os.path.join(root, file)))

# Combines all the data frames into one
df_aapl = pd.concat(df_list)
df_aapl.head()

Next let’s convert the date column to datetime objects and sort by date values since the csv files may not have been loaded in order.

df_aapl['Date'] = df_aapl['Date'].apply(lambda x: datetime.strptime(x, "%m/%d/%Y"))
df_aapl = df_aapl.sort_values("Date").reset_index(drop=True)

Now for the purposes of this project we only need the date and the close price columns, so let’s drop the others. We also want to get the time of the close price which is 4pm on each day, so let’s add 4pm to our current datetime values, and change the column name to Datetime.

from datetime import time

df_aapl = df_aapl.drop(columns=["Open", "High", "Low", "Volume"])
df_aapl["Date"] = df_aapl["Date"].apply(lambda x: datetime.combine(x, time(16, 0)))
df_aapl = df_aapl.rename(columns={"Date":"Datetime"})
df_aapl.head()

Ok now let’s load up our headlines csv file we created in Part 2

df_headlines = pd.read_csv("headlines_w_sentiment.csv")
df_headlines.head()

We want to convert the date and time text columns into a single datetime column of datetime objects, as we will need to know each headlines distance in time from the datetimes that we will be calculating our aggregated sentiment scores at.

df_headlines["Datetime"] = df_headlines[["Date", "Time"]].apply(lambda row: ",".join(row.values.astype(str)), axis=1)
df_headlines["Datetime"] = df_headlines["Datetime"].apply(lambda x: datetime.strptime(x, "%m/%d/%Y,%I:%M%p"))
df_headlines = df_headlines.sort_values("Datetime").reset_index(drop=True)
df_headlines = df_headlines.drop(columns=["Date", "Time"])

df_headlines.head()

Now we want to aggregate the headline sentiments to get sentiment values for each datetime in the stock price data frame. There are many different methods to aggregating sentiment scores. The most common method seems to be just to take the average of the sentiment scores over the past n days. We are going to go one step further and take a weighted sum of sentiment scores over the past n days using a gaussian function, so that the sentiment of more recent news headlines have higher weighting than articles further back in time, as would be the case in reality.

So let’s create a function that will return a Positive, Negative and Neutral sentiment value for any given time t, by taking a gaussian weighted sum of our headline sentiments over the past n days. I decided to set n as 7, to aggregate sentiments from headlines over the previous week, but you could set this to any amount of days that you want. If you really want to go overkill, you could test for different values of n and different values for the curve of the gaussian function to see which maximises correlation between the sentiment values and price movement, but I would really only go that far if your sentiment values have already proved to be useful and you want to maximise their usefulness.

from datetime import timedelta
import numpy as np
from scipy.stats import norm

n = 7
gaussian_curve = 1.0

# Gets sentiment values at time t by aggregating over last n days using gaussian weighted sum
def get_sentiment_at_time(t):
    prev_time = t - timedelta(days=n)
    relevant_headlines = df_headlines[(df_headlines["Datetime"] < t) & (df_headlines["Datetime"] > prev_time)].reset_index(drop=True)

    if(len(relevant_headlines) != 0):
        relevant_headlines["Weight"] = relevant_headlines["Datetime"].apply(lambda x: norm.pdf(gaussian_curve*(t - x).total_seconds()/(n*24*60*60)))
        pos_weighted_sum = (relevant_headlines["Positive"]*relevant_headlines["Weight"]).sum()
        neg_weighted_sum = (relevant_headlines["Negative"]*relevant_headlines["Weight"]).sum()
        neu_weighted_sum = (relevant_headlines["Neutral"]*relevant_headlines["Weight"]).sum()
    else:
        pos_weighted_sum = np.nan
        neg_weighted_sum = np.nan
        neu_weighted_sum = np.nan
    return pd.Series([pos_weighted_sum, neg_weighted_sum, neu_weighted_sum], index=["pos_sentiment", "neg_sentiment", "neu_sentiment"])

As you can see, I have dealt with the case where there are no headlines over the past n days by just returning None. When we use this function to get sentiment values for each market closing datetime, we can just replace any None values with the sentiment from the previous market day, as I think we can assume that if there have been no new articles, the sentiment will continue to be the same.

So now let’s run our function for each datetime in the AAPL stock price data frame and add the outputted scores to the data frame.

# Removing n days from beginning of stocks dataframe as we will be summing sentiments over previous n days
df_aapl = df_aapl[df_aapl["Datetime"] > (df_aapl["Datetime"][0] + timedelta(days=n-1))].reset_index(drop=True)

# Apply sentiment aggregation function to each closing time in stocks dataframe and add as new columns
sentiment_values = df_aapl["Datetime"].apply(lambda t: get_sentiment_at_time(t))
df_aapl = pd.concat([df_aapl, sentiment_values], axis=1)

df_aapl.head()

Now let’s replace any missing sentiment values with the value from the previous market day. I am using isna().sum() to make sure this works and there are no null values left.

for index, row in df_aapl.iterrows():

    if pd.isnull(row["pos_sentiment"]):
        df_aapl.iloc[index, 2:5] = df_aapl.iloc[index-1, 2:5]

df_aapl.isna().sum()

Ok great, we can see our dataframe now has no missing values.

Now we want to check if there is any correlation between the sentiment values we have calculated, and the price movement of the stock over the following day. For ease of access, let’s first make a column “Price Movement” that will contain the price movement over the following market day for each closing price.

price_movement = []

# Loops though all price values except for final value and get difference between this value and the value in the next row
for i in range(len(df_aapl)-1):
    price_movement.append(df_aapl["Close"][i+1] - df_aapl["Close"][i])
    
# Drops final row where price change couldn't be calculated
df_aapl = df_aapl.drop(df_aapl.index[-1])

df_aapl["Price Movement"] = price_movement
df_aapl.head()

Now there are a few ways we could check the correlation between the sentiments and price movement. The corr() function is the obvious choice, but I want to see a visual representation of this. So let’s plot the positive sentiment values againts the price movement with a linear regression line. If there is any correlation, you would expect to see high values of price movement for high values of positive sentiment, and a positive gradient on the regression line.

import seaborn as sns
import matplotlib.pyplot as plt
from scipy.stats import linregress

plt.figure(figsize=(20, 10))

regression_line = sns.regplot(x=df_aapl["pos_sentiment"], y=df_aapl["Price Movement"], scatter_kws={'marker': 'o'}, line_kws={'color': 'red'})
slope = linregress(df_aapl["pos_sentiment"], df_aapl["Price Movement"]).slope

# Print slope of regression line and correlation value 
print(f"Correlation value = {df_aapl['Price Movement'].corr(df_aapl['pos_sentiment'])}")
print(f"Slope (Gradient) of the Regression Line: {slope}")

plt.title('Price Movement for Positive Sentiment Values')
plt.xlabel('Positive Sentiment')
plt.ylabel('Price Movement')

plt.show()

Now let’s do the same for negative sentiment.

plt.figure(figsize=(20, 10))

regression_line = sns.regplot(x=df_aapl["neg_sentiment"], y=df_aapl["Price Movement"], scatter_kws={'marker': 'o'}, line_kws={'color': 'red'})
slope = linregress(df_aapl["neg_sentiment"], df_aapl["Price Movement"]).slope

# Print slope of regression line and correlation value 
print(f"Correlation value = {df_aapl['Price Movement'].corr(df_aapl['neg_sentiment'])}")
print(f"Slope (Gradient) of the Regression Line: {slope}")

plt.title('Price Movement for Negative Sentiment Values')
plt.xlabel('Negative Sentiment')
plt.ylabel('Price Movement')

plt.show()

The correlation is very low for both, and the price movement is pretty evenly distributed around 0 for all values of postive and negative sentiment. This essentially means that the sentiment found in our article headlines is not a good indicator of price movement.

But do not despair! This was to be expected, if it were this easy to predict stock price changes, we would all be rich. Sentiments from Investopedia headlines may not be a useful indictor of stock price movement, but it might still be a useful addition to a more complex indicator taking data from many sources. There are also many paid API’s with higher quality sources of financial news, Investopedia was just a good free example for this project.
